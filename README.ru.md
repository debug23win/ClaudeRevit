# Claude Revit

[English](README.md) | **Русский**

Claude AI внутри Autodesk Revit — закрепляемая панель чата со **153 инструментами**, через которые Claude напрямую читает и изменяет вашу модель. Попросите его построить стены, создать спецификации, расставить семейства, образмерить оси, заармировать конструкции, собрать параметрическое семейство, оформить листы — и он сделает это сам. Работает на **Revit 2025, 2026 и 2027**.

---

## Возможности

- **Закрепляемая панель чата** в Revit с потоковым выводом ответов
- **153 инструмента**: моделирование, виды, листы, аннотации, спецификации, фильтры, семейства, редактор семейств, армирование
- **Несколько провайдеров ИИ** — Claude (Sonnet 5 / Opus 4.8 / Fable 5 / Haiku 4.5, + Sonnet 4.6 / Opus 4.7) **или** любой OpenAI-совместимый эндпоинт: DeepSeek, Google Gemini, ChatGPT/OpenAI, Qwen, OpenRouter, Groq, локальные **Ollama** / **LM Studio**. Выберите «Alt» в списке моделей; бесплатным и локальным моделям ключ Anthropic не нужен.
- **Одна отмена на запрос** — Ctrl+Z откатывает всё, что Claude сделал за один ход
- **Понимает выделение** — зелёная плашка показывает, что выделено; Claude знает, что значит «это»
- **Markdown** и **выделяемый текст** в сообщениях
- **Кликабельные ID элементов** — клик по ID в результате инструмента выделяет и показывает элемент в модели
- **Учёт стоимости и баланса** с кэшированием промпта (TTL 1 час на системный промпт и инструменты — каждый ход оплачивает только новые токены); введите свой баланс, и панель покажет остаток
- **Полная память контекста** — вся история диалога (вызовы инструментов, результаты, ID элементов) переживает перезапуск Revit: Claude помнит, что он построил
- **Автокомпакция** — при превышении контекстного бюджета модели старые ходы сжимаются в резюме, не переполняя окно
- **Старение результатов инструментов** — старые результаты усекаются на месте (и архивируются) ради экономии токенов; `get_full_result` достаёт полный результат по запросу
- **Локальное обучение** — `save_memory` хранит ваши предпочтения и стандарты проекта; каждый запуск кода журналируется вместе с изменениями модели, проверенные приёмы попадают в системный промпт и **переживают очистку чата**, а при закрытии Revit (или по команде `generate_diagnostic_report`) формируется диагностический отчёт о повторяющихся скриптах — чтобы превратить их в отдельные инструменты
- **Доступ ко всему Revit API (по согласию)** — когда готового инструмента нет, Claude может выполнить код: `execute_csharp` (по умолчанию — компилируемый C# в управляемой транзакции) или `run_dynamo_python` (для Python-фрагментов). **По умолчанию выключено**: включается галочкой в настройках.
- **Настраиваемый лимит вызовов** — сколько раундов инструментов Claude может сделать на одно сообщение (по умолчанию 24), поднимается в настройках для длинных автоматических задач
- **Подтверждение опасных операций (опция)** — по умолчанию выключено (каждый ход и так одна отмена); при желании включается диалог «Разрешить/Запретить»
- **API-ключ вводится в панели** (шестерёнка) и хранится зашифрованным (Windows DPAPI) — без открытых переменных окружения

## Установка (для пользователей)

Понадобятся:

- **Autodesk Revit 2025, 2026 или 2027** — установщик определяет, какие из них у вас стоят, и даёт отметить нужные галочками
- **Windows** (Revit работает только на Windows)
- **API-ключ Anthropic** — получите на [console.anthropic.com](https://console.anthropic.com/settings/keys) и пополните баланс в **Billing** — *или* бесплатный/альтернативный провайдер (DeepSeek, Gemini, OpenRouter, Groq, локальные Ollama / LM Studio…), настраивается в разделе Settings

Любой из двух способов:

### Вариант А — установщик .exe (проще всего)

Скачайте **`ClaudeRevit-Setup-vX.Y.exe`** из [последнего релиза](https://github.com/debug23win/ClaudeRevit/releases/latest), запустите, отметьте нужные версии Revit → Install. Готово.

> Windows SmartScreen при первом запуске может показать «Windows защитила ваш компьютер» (установщик пока не подписан). Нажмите **Подробнее → Выполнить в любом случае**.

### Вариант Б — одна команда PowerShell

```powershell
iwr https://raw.githubusercontent.com/debug23win/ClaudeRevit/main/install.ps1 | iex
```

В обоих случаях: запустите Revit, откройте любой проект и найдите вкладку **Claude** в ленте. Кнопка **Chat** открывает панель справа. При первом запуске нажмите **⚙** и вставьте API-ключ (или настройте альтернативного провайдера).

Для обновления просто повторите установку — подтянется последний релиз.

## Сборка из исходников (для разработчиков)

Понадобятся:

- **Visual Studio 2026 Community** (или Rider, или VS Code с C# Dev Kit)
- **.NET 8 SDK** и **.NET 10 SDK** — 2025/2026 собираются под .NET 8, 2027 — под .NET 10
- **Autodesk Revit** локально (нужен только для отладки по F5 — компиляция работает и без него)

Шаги:

1. Клонировать:
   ```powershell
   git clone https://github.com/debug23win/ClaudeRevit.git
   cd ClaudeRevit
   ```
2. Открыть `ClaudeRevit.sln` в Visual Studio.
3. ПКМ по проекту → **Restore NuGet Packages**.
4. Назначить `ClaudeRevit` стартовым проектом.
5. Нажать **F5**.

Пост-сборочный шаг сам копирует DLL и манифест в `%AppData%\Autodesk\Revit\Addins\<год>\`. F5 запускает Revit с подключённым отладчиком.

### Как устроена сборка

- Ссылки на Revit API берутся из NuGet-пакетов **Nice3point.Revit.Api.RevitAPI** и **RevitAPIUI** — локальный Revit для компиляции не нужен.
- Пост-сборочный таргет `DeployToRevit` копирует в `%AppData%\Autodesk\Revit\Addins\<год>\`. Чтобы пропустить (например, на CI): `-p:SkipDeploy=true`.
- Отдельный таргет `PackageRelease` складывает артефакты релиза в `bin\Release\release\` — его использует GitHub Actions.

## Версии Revit

Плагин поддерживает **Revit 2025, 2026 и 2027** и собирает отдельную сборку для каждой,
потому что версии работают на разных рантаймах .NET (2025/2026 → .NET 8, 2027 → .NET 10).
При установке обнаруженные версии уже отмечены; отметьте нужные, и каждая получит свою
сборку в отдельной папке `%AppData%\Autodesk\Revit\Addins\<год>\`.

Собрать локально под конкретную версию — параметром `RevitVersion`:

```powershell
dotnet build ClaudeRevit\ClaudeRevit.csproj -c Release -p:RevitVersion=2026
```

Пакет Nice3point (`$(RevitVersion).0.*`), целевой фреймворк и папка деплоя следуют за этим
одним значением. Различия API между версиями закрыты символами компиляции
`REVIT2025` / `REVIT2026` / `REVIT2027`. Revit 2024 и раньше работают на .NET Framework 4.8
и в текущую сборку не входят.

## Выпуск новой версии

Workflow `.github/workflows/release.yml` собирает все три версии Revit и публикует релиз.
Запустите его на вкладке Actions (**Run workflow** → введите версию, например `v1.30`) или
запушьте тег `v*`. GitHub Actions:

1. Собирает каждую версию Revit (2025/2026/2027) в Release с `SkipDeploy=true`
2. Делает zip на каждую версию (`ClaudeRevit-vX.Y-Revit<год>.zip`)
3. Компилирует установщик Inno Setup, объединяющий все три, со страницей выбора версий
4. Публикует GitHub Release с установщиком и zip-архивами

Пользователи получают новую версию тем же установщиком / командой `install.ps1`.

## Инструменты

Плагин даёт Claude **153 инструмента** по категориям:

- **Чтение модели** — элементы, параметры, уровни, материалы, стадии, семейства, информация о проекте, предупреждения, пакетное чтение позиций и габаритов (мм)
- **Создание геометрии** — стены, перекрытия, крыши, помещения, уровни, оси, двери, окна, колонны, балки, фундаменты, MEP (воздуховоды/трубы), топография, витражи
- **Армирование** — ряды стержней (с количеством и шагом), площадное и по траектории, защитные слои, список типов, инспекция существующего армирования
- **Редактор семейств** — нативная сборка параметрических семейств: список/добавление/удаление параметров, формулы, значения (мм), переключение экземпляр/тип, привязка параметров вложенных элементов, линейные массивы с параметрическим количеством, образмеривание между ссылками с меткой-параметром
- **Обучение и запасной выход** — `save_memory`, `get_script_journal`, `generate_diagnostic_report`; `execute_csharp` / `run_dynamo_python` (код для действий, не покрытых инструментами — по умолчанию выключено, включается галочкой)
- **Операции с элементами** — перемещение, поворот, копирование, зеркало, массив, удаление, параметры, закрепление, соединение/разъединение
- **Виды** — 3D, планы, разрезы, фасады, фрагменты, дублирование, зависимые виды, масштаб, шаблоны, подрезка/секущий бокс
- **Листы** — создание листов, размещение видов и спецификаций, перемещение видовых экранов
- **Спецификации** — настоящие ViewSchedule с выбором полей, экспорт CSV
- **Аннотации** — размеры, марки, текст, линии детализации, штриховки, опорные плоскости, ревизии, высотные отметки
- **Видимость** — скрыть/изолировать, цветовые переопределения, видимость категорий, фильтры
- **Интерактив** — `pick_point_in_view` для сценариев «кликни, куда поставить»
- **Семейства** — загрузка .rfa, список загруженных, размещение экземпляров, дублирование/правка типов
- **Группы** — создание, размещение, разгруппировка
- **Экспорт** — изображения видов, PDF, DWG, CSV спецификаций, сохранение документа
- **Выделение** — `select_similar` («выделить все экземпляры этого типа»)

Полный список — в папке [`ClaudeRevit/Tools/`](ClaudeRevit/Tools): один `.cs`-файл = один инструмент.

## Архитектура

- **`App.cs`** — точка входа `IExternalApplication`: регистрирует инструменты, панель, кнопку в ленте и хук обучения DocumentChanged
- **`Tools/`** — классы `IRevitTool`. Новый файл + строка регистрации в `App.cs` → инструмент доступен Claude
- **`Tools/ToolDispatcher.cs`** — `IExternalEventHandler`: исполняет вызовы в API-потоке Revit; каждый инструмент — в `Transaction`, весь ход — в `TransactionGroup` (одна отмена на запрос)
- **`Services/ChatService.cs`** — агентский цикл, независимый от провайдера: стримит каждый ход либо через Anthropic API, либо через OpenAI-совместимый бэкенд в общую модель хода; кэширование промпта, инструментов и истории (TTL 1 час), компакция и старение результатов
- **`Services/OpenAIBackend.cs`** — SSE-стриминг, вызов функций и учёт токенов для любого OpenAI-совместимого эндпоинта (модель «Alt»)
- **`Services/ScriptJournal.cs` / `ExperienceStore.cs`** — слой обучения: журнал запусков кода с изменениями модели, дайджест проверенных приёмов в системный промпт, диагностический отчёт
- **`Services/HistoryStore.cs` / `MemoryStore.cs` / `ApiKeyStore.cs` / `SettingsStore.cs`** — персистентность: история диалога, память Claude, зашифрованные ключи, настройки
- **`UI/ChatPaneView.xaml`** — WPF-панель чата (виртуализированный список сообщений, markdown, кликабельные ID)
- **`Services/SelectionService.cs`** — отслеживание выделения, чтобы контекст Claude был актуальным
