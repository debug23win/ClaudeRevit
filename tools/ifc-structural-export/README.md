# IFC → аналитическая модель (DXF/DWG)

`ifc_to_analytical.py` извлекает из IFC-модели **несущие элементы** и
экспортирует упрощённую **расчётную (аналитическую) модель** в DXF (и,
опционально, DWG) — в виде осей и срединных плоскостей, а не тел:

| Элемент IFC            | Категория | Выход в DXF                              | Слой        |
|------------------------|-----------|------------------------------------------|-------------|
| `IfcColumn`            | колонны   | ось — отрезок `LINE`                     | `S_COLUMNS` |
| `IfcBeam` / `IfcMember`| балки     | ось — отрезок `LINE`                     | `S_BEAMS`   |
| `IfcSlab` / `IfcPlate` | плиты     | контур срединной плоскости `POLYLINE`    | `S_SLABS`   |
| `IfcWall*`             | стены     | вертикальная панель по оси × высота      | `S_WALLS`   |

Дополнительно: точки-узлы (концы стержней) на слое `S_NODES` и текстовая
подпись со смещением координат на слое `S_INFO`.

## Идея

Вместо триангуляции тел скрипт читает **параметрическое** представление
элемента и сводит его к аналитическому примитиву:

```
IfcExtrudedAreaSolid  →  профиль + оси + направление + глубина
    линейный элемент (колонна, балка)  →  линия через центр профиля
                                          от низа выдавливания до низ+глубина
    плита                              →  контур профиля, смещённый
                                          на половину толщины (срединная плоскость)
    стена                              →  ось в плане (Axis) × высота стены
```

Обёртки `IfcMappedItem` (типовые представления Revit) и булевы операции
`IfcBooleanClippingResult` (вырезы) разворачиваются рекурсивно. Элементы с
телом-`Brep` или иным неразрешимым представлением обрабатываются
запасным способом — приведением триангулированной сетки к
ориентированному габаритному ящику (OBB через PCA): длинная ось → линия,
тонкий габарит → срединная плоскость.

## Координаты

Revit экспортирует модель в реальных геодезических координатах (в этом
файле — порядка 1349 км от начала). Поэтому по умолчанию геометрия
**переносится** в ноль по минимуму габаритного ящика; применённое смещение
пишется в DXF (слой `S_INFO`) и выводится в консоль, чтобы модель можно
было вернуть на место. Отключается флагом `--no-recenter`.

## Установка

```bash
pip install -r requirements.txt      # ifcopenshell, ezdxf, numpy
```

## Запуск

```bash
# колонны + балки + плиты (по умолчанию), результат model.dxf
python ifc_to_analytical.py model.ifc

# все несущие элементы, свой выходной файл
python ifc_to_analytical.py model.ifc -o analytical.dxf \
       --elements columns,beams,slabs,walls

# в метрах, без переноса координат
python ifc_to_analytical.py model.ifc --unit m --no-recenter

# дополнительно сохранить DWG (нужен ODAFileConverter в PATH)
python ifc_to_analytical.py model.ifc --dwg
```

### Параметры

| Флаг            | Значение                                                        |
|-----------------|-----------------------------------------------------------------|
| `-o, --output`  | выходной DXF (по умолчанию `<ifc>.dxf`)                          |
| `--elements`    | список из `columns,beams,slabs,walls` (по умолчанию первые три)  |
| `--unit`        | единицы вывода `mm` / `cm` / `m` (по умолчанию `mm`)             |
| `--no-recenter` | сохранить исходные (геодезические) координаты                    |
| `--dwg`         | дополнительно сконвертировать в DWG через ODAFileConverter       |

## DWG

`ezdxf` пишет только DXF. Для DWG скрипт вызывает бесплатный
**ODA File Converter** (Open Design Alliance), если он есть в `PATH`.
DXF без изменений открывается в AutoCAD, nanoCAD, BricsCAD, Revit,
ЛИРА-САПР, SCAD, Robot и т. п., поэтому DWG обычно не требуется.

## Проверка на реальной модели

На тестовой модели (IFC2X3, Revit 2027, здание ~36 × 81 × 11 м):

```
columns    32   balки 161   slabs 50   walls 56
fallback   14   (Brep/CSG сведены к OBB)
```

Все колонны — вертикальные оси; плиты — горизонтальные контуры;
стены — вертикальные панели на высоту стены; координаты перенесены в ноль.
